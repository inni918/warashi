# Warashi

> 無料・オープンソースで、はじめての人にもやさしい **デスクトップ AI コンパニオン**。Live2D アバター付きで、長期記憶・自発的なおしゃべり・自然な音声・おやすみモードを備えています。LLM（頭脳）だけ自分で用意すれば、あとは最初から全部そろっています。

**言語:** [English](./README.md) | [繁體中文](./README.md#繁體中文) | **日本語**

## これは何？

**Warashi** は、画面の中の Live2D キャラクターを「本当に会話できる AI コンパニオン」に変えてくれます。あなたのことを覚えていて、自分から話しかけてきて、あなたが話している間は耳をかたむけ、「おやすみ」と言えば静かになります。

これは、すばらしいオープンソースプロジェクト [Open-LLM-VTuber](https://github.com/Open-LLM-VTuber/Open-LLM-VTuber) を **やさしく再パッケージ** したものです。私たちはその土台の上に立っています。上流（Open-LLM-VTuber）が、しっかりした Live2D + 音声認識/音声合成 + LLM の仕組みを提供してくれているので、この fork ではそれを「**ダウンロード → ダブルクリック → おしゃべり**」という、技術にくわしくない人でも使える体験に包み直しました。さらに、記憶システム、自発的な会話、自然に割り込める音声、キャラクター管理、アプリ内のセットアップウィザード、そして完全な二言語（英語 / 繁體中文）UI を追加しています。

**私たちの方針 — そして、あえて「やらない」こと：**

- **無料・オープンソース、寄付は任意。** 有料プランも、ペイウォール（課金の壁）もありません。
- **モバイル版は作りません。** デスクトップ専用です（macOS / Windows）。
- **モデルのマーケットプレイスや、著作権のあるキャラクターの同梱はしません。** これは Live2D の商用ライセンスの落とし穴を避けるためです。私たちが同梱するのは中立的な無料デフォルトだけ。キャラクター・声・LLM はあなた自身が用意します。

> 本プロジェクトは Open-LLM-VTuber の上に構築されています。完全なクレジットと各コンポーネントのライセンスは [`NOTICE`](./NOTICE) を、元プロジェクトのドキュメントは [`README.upstream.md`](./README.upstream.md) をご覧ください。

## 機能

- **長期記憶** — あなたが誰で、何に取り組んでいるかを覚えていて、時間とともにあなたのことを少しずつ理解していきます。キャラクターごとに用意された「コア記憶」がペルソナに注入され、各ターンのあと LLM が「何を残す価値があるか」を判断します。更新はすぐに反映されます（再起動不要）。記憶の上限は調整できます。
- **深い記憶の呼び出し（FTS5 trigram）** — オプトイン式で、これまで話したすべての履歴を全文検索できます。CJK（中日韓）にもきちんと対応しています。
- **自発的な話題** — しばらく沈黙が続くと、自分から話題をふってきます。任意で、最新の AI / テック / アニメ / ゲームのニュースを取り込んで話題にすることもできます（純粋な標準ライブラリだけのヘルパーで、**API キーは不要**）。
- **自然に割り込める音声チャット** — いつでも話しかけられます。マイクの順番を待つ必要はなく、本物の会話のように途中で言葉をさえぎることもできます。
- **スリープ / おやすみ（おじゃましない）モード** — 「晚安（おやすみ）」と言えば自分から話しかけてこなくなり、次にあなたが話しかけたときに再開します。キーワードは自由に変えられます。
- **キャラクター管理** — キャラクターの作成 / 編集 / 切り替え / 削除ができます。名前 + ペルソナ + Live2D の見た目 + 声 + キャラごとに独立した記憶を持てます。
- **初回起動時のセットアップウィザード** — API キー（OpenAI / Claude / Gemini）を貼り付けるか、ローカルの Ollama モデルを選びます。ウィザードは保存する前に、軽いテスト呼び出しを実行して動作を確認します。
- **LLM 設定タブ** — API キーを貼り付けるか、Ollama モデルを選ぶ／手入力します（ローカルモデル、または Ollama 経由で提供されるクラウドモデル）。
- **パフォーマンスプリセット** — ライト / スタンダード / ハイパフォーマンスの 3 段階。ASR/TTS エンジンの選択 + 記憶の整理（consolidation）の頻度 + モデルの常駐（keep-alive）をまとめて切り替えます。
- **言語をまたぐ翻訳** — 任意の字幕 / 音声の翻訳（デフォルトはオフ）。
- **すぐに使える** — サンプルの Live2D モデル + 無料のクラウド TTS（edge-tts）+ 自動ダウンロードされる音声認識（ASR）モデル（約 1GB。初回起動時に一度だけダウンロードされ、数分かかります）が同梱されています。あなたは LLM を差し込むだけです。
- **完全な二言語 UI** — 繁體中文（zh）と英語（en）。

## クイックスタート（ダウンロード → ダブルクリック → おしゃべり）

いちばん簡単な道のり。**ターミナルは不要です。**

> **始める前に：AI の「頭脳」（LLM）が必要です。**
> Warashi は **体と顔** です — アバター、声、記憶。実際に考えて話す **頭脳** は、*あなた自身* が用意する別の AI です。初回起動のウィザードで設定します。選択肢は、簡単な順に次のとおりです。
> - **（おすすめ — 無料・プライベート・あなた自身のパソコンで動く）Ollama によるローカルモデル。** 無料の **[Ollama](https://ollama.com)** アプリをインストールし、ターミナルで `ollama pull qwen2.5:3b` を実行します（約 1.9 GB の小さなモデル）。Warashi のデフォルトがすでにこれを指しているので、そのまま動きます — **アカウント不要、API キー不要、費用ゼロ、オフラインでも動き、あなたの会話がパソコンの外に出ることはありません。** ふつうの 8〜16 GB のノートパソコンで問題なく動きます。（もっと賢い返事がほしくて RAM に余裕がある？ `qwen2.5:7b` のような大きめのモデルを pull して、設定で選んでください。）
> - **（任意 — パソコンが非力なら品質が上がる）Ollama Cloud の無料プラン。** Ollama は *Ollama 側* のサーバーで、より大きなモデルを無料で動かせます（上限あり）。無料アカウントが必要です — 下の **方法 B** を参照。先にそのクラウドモデルを `ollama pull` する必要があります。
> - **（任意 — 無料の中ではいちばん高品質）無料のホスト型 API キー。** Google AI Studio（Gemini）、Cerebras、Groq が無料のキーを提供しています（クレジットカード不要）。無料の選択肢の中ではいちばん高品質ですが、アカウント + キーが必要で、会話はその提供元に送られます。**方法 C** を参照。
> - **（すでに有料で契約しているなら）クラウドの API キー**：OpenAI / Claude / Gemini のもの — 最高品質で、1 回のおしゃべりで数円程度。**方法 D** を参照。

1. **Warashi をダウンロードする。** [**Releases ページ**](https://github.com/inni918/warashi/releases/latest) にアクセスし、最新の `Warashi-*.zip` をダウンロードして解凍します（例：デスクトップへ）。_（または、リポジトリのトップページで緑色の **`<> Code`** ボタン → **Download ZIP** からでも入手できます。）_
2. 解凍したフォルダの中の **ランチャーをダブルクリック** します：
   - **macOS：** `start-companion.command`
   - **Windows：** `start-companion.bat`
   - 初回起動では、すべて（まず `uv`、続いて依存関係）が自動でインストールされ、数分かかることがあります。**その窓は閉じないでください — それがサーバー本体です。**
3. ブラウザが自動で **http://localhost:12393** を開きます。初回は **セットアップウィザード** が表示されます。**API キーを貼り付ける**（OpenAI / Claude / Gemini）か、**ローカルの Ollama モデルを選ぶ** かのどちらかです。ウィザードは保存する前に選んだ内容をテストします。

   ![Warashi 初回起動時のセットアップウィザード](assets/warashi-setup.png)

   *初回起動時のセットアップウィザード。ここで AI の「頭脳」を差し込みます。*

4. **再起動すると、新しい頭脳が有効になります。** **手順 2 で開いたランチャー / ターミナルの窓を閉じて** 終了し（これでサーバーが止まります）、もう一度 **ランチャーをダブルクリック** して、新しい LLM とともに起動し直します。（アプリ内でも、LLM の変更は「再起動後 — またはキャラクターを一度切り替えたあとに反映されます」と案内されます。）あとはおしゃべりを始めましょう。ページを一度クリックすると音声が有効になります。

> **macOS の Gatekeeper（初回のみ）：** ダブルクリックすると「*開発元が未確認のため開けません*」と表示されることがあります。これは署名されていないオープンソースアプリでは正常なことです。`start-companion.command` を **右クリック → 開く → 開く** をダイアログで選んでください。一度許可すれば、それ以降はダブルクリックで開けます。（署名 / 公証（notarize）済みのビルドは提供していません — これが無料版です。）

> **Windows の SmartScreen（初回のみ）：** ダブルクリックすると、青い「**WindowsによってPCが保護されました**」というボックスが出ることがあります。これは署名されていないオープンソースアプリでは正常なことです。**詳細情報** → **実行** をクリックしてください。一度許可すれば、もう聞かれません。

そのままの状態では、同梱の **mao** サンプル Live2D モデルと **edge-tts**（無料のクラウド音声、GPU 不要）が使われます。初回起動時には、音声認識（speech-to-text）モデルも自動でダウンロードされます —— これは約 **1GB** あるため、**いちばん最初の起動では、一度きりのダウンロードと展開に数分かかることがあります**。その間、ランチャーの窓は固まったように見えますが、固まってはいません。**閉じずに、終わるまで待ってください**。これが起きるのは最初の一回だけです。

### ターミナルのほうがいい？（上級者向け）

ほとんどの人は、上の **Download ZIP** の道のりで十分です。ターミナルに慣れているなら、代わりにリポジトリを clone してもかまいません。**Python ≥ 3.10, < 3.13** と [`uv`](https://github.com/astral-sh/uv) が必要です。

```bash
git clone https://github.com/inni918/warashi.git && cd warashi
uv sync                  # 依存関係をインストール
uv run run_server.py     # サーバーを起動
# http://localhost:12393 を開く → セットアップウィザード → おしゃべり
```

ウィザードが、あなたの LLM の選択を `conf.yaml` に書き込んでくれます。もちろん手で編集することもできます（下記参照）。

## 起動・終了・自動起動

**起動するには（「ホスト」を開く）：**

- **Windows：** **`start-companion.bat`** をダブルクリック
- **macOS：** **`start-companion.command`** をダブルクリック —— 初回は macOS にブロックされることがあるので、ファイルを右クリック → **開く** → もう一度 **開く**。以降は普通のダブルクリックでOKです。

黒いコマンドウィンドウが開きます —— **そのウィンドウが本体（サーバー）です。会話している間は開いたままにしてください。** 準備ができると、自動でブラウザにアプリ（`http://localhost:12393`）を開きます。イメージとしては、**コマンドウィンドウがエンジン、ブラウザのタブは画面だけ** です。

> 初回起動のみ：約 1GB の音声モデルをダウンロードするため数分かかります。ウィンドウが固まったように見えても問題ありません —— 開いたまま完了を待ってください。

**終了するには：**

- **黒いコマンドウィンドウを閉じる**（またはその中で **Ctrl + C**）と、完全に停止します。
- **ブラウザのタブ**を閉じても画面が隠れるだけで、サーバーは動き続けます。本当に止めるにはコマンドウィンドウを閉じてください。
- ローカル頭脳用に **Ollama** を入れている場合、バックグラウンドで動き続けます。アイドル時は軽いのでそのままでもOK。止めたいときはタスクトレイ（Windows）／メニューバー（macOS）のアイコンから Quit してください。

**また開くには：** 同じランチャーをもう一度実行するだけです（これが日々の起動方法です）。

**パソコンの起動時に自動で立ち上げる（任意）：**

- **Windows**
  1. **`start-companion.bat`** を右クリック → **ショートカットの作成**。
  2. **Win + R** を押し、**`shell:startup`** と入力して **Enter** —— スタートアップ フォルダーが開きます。
  3. ショートカットをそのフォルダーに入れます。これでログインのたびに自動起動します。*（取り消すには、そのフォルダーからショートカットを削除。）*

- **macOS**
  1. **システム設定 → 一般 → ログイン項目と機能拡張** を開く。
  2. **ログイン時に開く** の下で **+** をクリックし、**`start-companion.command`** を選ぶ（またはファイルをリストにドラッグ）。
  *（取り消すには、選んで **−** をクリック。）*

どちらの方法でも、ログイン時にコマンドウィンドウ（続いてブラウザ）が自動で開きます。補足：起動のたびにターミナルのウィンドウが出るのは正常です（エンジンが起動しているだけ）。ローカルの Ollama 頭脳を使う場合、Ollama はインストール後に自動起動するので、全体がひとりでに立ち上がります。

## LLM のセットアップ（必須）

クラウド LLM の API キー、**または** 動作中のローカル LLM の、**どちらか一方** が必要です。コンパニオンとのおしゃべりには安いモデルで十分で、最上位のモデルは必要ありません。

#### 方法 A — ローカルの Ollama（おすすめ：無料・プライベート・アカウント不要）
[Ollama](https://ollama.com/download) をインストールし、ターミナルで `ollama pull qwen2.5:3b` を実行します（約 1.9 GB）。Warashi のデフォルトはすでに `qwen2.5:3b` を使っているので、ダウンロードが終われば次回の再起動後にそのまま動きます — API キー不要、アカウント不要、クラウド料金なし、完全オフライン、そして会話はあなたのパソコンに留まります。ふつうの 8〜16 GB のノートパソコンで快適に動きます。もっと賢い返事がほしいときは、大きめのモデル（例：`qwen2.5:7b`）を pull して、LLM 設定タブで設定してください。

#### 方法 B — Ollama 経由でクラウドモデルを使う（無料アカウント；パソコンが非力なら有効）
Ollama は自前のサーバーで *より大きな* モデルを動かせるので、遅いパソコンでもよい返事が得られます。無料プランですが、アカウントが必要で、先にモデルを pull する必要があります：
1. [ollama.com/download](https://ollama.com/download) から Ollama をインストールします（v0.12+）。
2. [ollama.com](https://ollama.com) で無料アカウントを作り、ターミナルで `ollama signin` を実行します。
3. **`ollama pull gpt-oss:20b-cloud` を実行します — 動かす前に必ず pull する必要があります。**（設定で名前を入力するだけでは足りません。）
4. LLM 設定タブで Ollama を選び、モデルを `gpt-oss:20b-cloud` に設定します。

`gpt-oss:20b-cloud` は無料プランに向いた、いちばん軽いモデルです。`qwen3.5:cloud` や `minimax-m3:cloud` はより強力ですが、無料の上限をより速く使い切ります。

> **「無料」について正直に：** クレジットカード不要で $0、無料アカウントだけで使えます — ただし *軽い利用向け* のプランです：一度に 1 つのクラウドモデルまで、セッション上限は約 5 時間ごとにリセット、加えて週次上限があり、Ollama は正確な数値を公開していないため、たくさん話すとリセットまで上限に達することがあります。推論は **Ollama のサーバー上** で動くので、完全に非公開にしておきたい内容は送らないでください。クラウドモデルはプレビュー段階です — **頼りにする前に、一度ちゃんと返事をするか確認してください。**

#### 方法 C — 無料のホスト型 API キー（Gemini / Cerebras / Groq）
無料の選択肢の中ではいちばんチャット品質が高いです。無料アカウントを作り（クレジットカード不要）、API キーを作成して、LLM 設定タブで対応する base URL とともに貼り付けます：
- **Google AI Studio（Gemini）：** `https://generativelanguage.googleapis.com/v1beta/openai/` — 無料枠が大きめ。Google が無料枠の会話を製品改善に使う場合がある点に注意。
- **Cerebras：** `https://api.cerebras.ai/v1` — 非常に高速、1 日あたり約 100 万トークン無料（無料枠では context window が短い）。
- **Groq：** `https://api.groq.com/openai/v1` — 非常に高速、1 日あたりのトークン上限あり。

各提供元のドキュメントにある最新のモデル名を使ってください。会話はその提供元に送られ、無料枠にはレート制限があります。

#### 方法 D — 有料のクラウド API キー（OpenAI / Claude / Gemini）
すでに契約しているなら、ウィザードにキーを貼り付けるだけです。最高品質で、小さなモデルならふつうは 1 回のおしゃべりで数円程度です。

### ⚠️ 推論（「思考」）モデルは非対応です

**`glm-4.7:cloud`** のような推論モデルは、答えを別の `reasoning` フィールドに入れ、通常の `content` フィールドを **空のまま** にします。このアプリは `content` しか読まないため、推論モデルでは **返事が空白** になります。そして読み上げる文字がないので、**音声も出ません**。

**おすすめ：** 通常の（推論ではない）チャットモデルを選んでください。小さくて速いモデルのほうが、結局のところより自然で、遅延の少ないコンパニオンになります。

> 手動編集する場合：LLM の設定は `conf.yaml` の `character_config → agent_config → llm_configs → openai_compatible_llm` の下にあります。ファイル内のコメントに、OpenAI / Claude / Gemini を自分のキーで指す方法が書いてあります。編集後はランチャーを再起動してください。

## その他の設定

### 記憶（コア + 深い呼び出し）
デフォルトでオンです。各キャラクターは自分の記憶を `chat_history/<conf_uid>/core_memory.md` に保持します。ペルソナに注入されるコア記憶に加えて、ターンごとに LLM が整理（consolidation）します（何を残すかはモデルが判断）。オプトインの **FTS5 trigram による深い呼び出し** は、長期的な記憶が欲しいときに、全履歴を検索します。記憶の上限は設定で調整できます。

### キャラクター
アプリ内でキャラクターの作成 / 編集 / 切り替え / 削除ができます。それぞれが自分の名前、ペルソナ、Live2D の見た目、声、そして **独立した記憶** を持ちます。自分の Live2D モデルを追加するには、`live2d-models/<name>/` の下に置き、`model_dict.json` にエントリを追加して、選択します。**著作権のあるキャラクターモデルを公開リポジトリに commit しないでください。**

#### もっとキャラクターを（任意）
ライセンス上の安全のため、Warashi が同梱しているのは **3 体の無料 Live2D オリジナルキャラクター**（`mao_pro`、`haru`、`hiyori`）だけです。もっと欲しい場合 — 男性執事キャラクターの **Natori（名取）** を含め — 公式ページから無料の公式 Live2D サンプルモデルを自分でダウンロードして入れることができます。**[Live2D のサンプルモデルページ](https://www.live2d.com/en/learn/sample/)** から、Live2D 自身のライセンスのもとで入手してください — 私たちは再配布していません。手順は [`docs/add-live2d-character.md`](docs/add-live2d-character.md) を参照してください。

### パフォーマンスプリセット
**ライト / スタンダード / ハイパフォーマンス** のプリセットが、ASR/TTS エンジンの選択、記憶の整理（consolidation）の頻度、モデルの常駐（keep-alive）をまとめて切り替えます。控えめなマシンならライトを、ハードウェアに余裕があればハイパフォーマンスを選びましょう。

### 自発的な話題 & ニュース
コンパニオンは、しばらく操作がないと話題をふってきます。任意で、同梱のニュースヘルパー（`scripts/news_topics.py`）で最新の見出しを取り込んで話題を更新できます — 純粋な標準ライブラリのみ、API キー不要 — そして、これを定期実行（cron / launchd / タスク スケジューラ）に登録しておくとよいでしょう（例：数時間おき）。

### スリープ / 静かモード
「晚安（おやすみ）」と言えば自分から話しかけてこなくなり、次のメッセージで再開します。キーワードは変更できます。

### 翻訳
任意の、言語をまたぐ字幕 / 音声の翻訳です。**デフォルトはオフ**（`conf.yaml` の `tts_preprocessor_config → translator_config`）。

### 音声
デフォルトは **edge-tts**（無料、ハードウェア不要）です。高品質なローカル / カスタム音声が欲しい場合は、[GPT-SoVITS](https://github.com/RVC-Boss/GPT-SoVITS) をサービスとして動かし、設定でそこを指してください（GPU または Apple Silicon が必要）。実在の人物の声をクローンすることは、あなた自身の法的責任です。

## トラブルシューティング

**アプリが開かない／黒いウィンドウがすぐ閉じる。**
たいていは、古いビルドが回復できない設定・エンジンの選択が原因です。**最新リリース**をダウンロードしてください（古い設定のままでも開きます）。更新したくない場合：アプリのフォルダーの `conf.yaml` を開き、`asr_model: 'faster_whisper'` を `asr_model: 'sherpa_onnx_asr'` に変えて保存し、再起動します。（`conf.yaml.backup` は復元しないでください — 同じ壊れた設定が入っています。）

**文字では返ってくるのに声が出ない。**
最新リリースに更新し（Windows に必要な音声ツールを同梱しています）、ランチャーをもう一度実行して依存関係を更新してください。音声（TTS）エンジンを GPT-SoVITS に切り替えた場合は別のサービスが必要です —— 設定 → パフォーマンス で **Edge TTS** に戻すと無料の内蔵音声が使えます。

**初回起動でブラウザに「このサイトにアクセスできません／接続が拒否されました」と出る。**
サーバーがまだ起動中です —— 初回は約 1GB の音声モデルをダウンロードするため数分かかります。黒いウィンドウは開いたままにし、起動した旨が表示されたら `http://localhost:12393` を再読み込みしてください。最新リリースは準備ができてから自動でブラウザを開きます。

**Windows：「WindowsによってPCが保護されました」（SmartScreen）、または何も起きないように見える。**
**詳細情報 → 実行** をクリックしてください —— これはランチャーであり、ウイルスではありません（署名がないだけ）。ウイルス対策ソフトが一度きりの `uv` インストールをブロックした場合は、このアプリを許可してもう一度実行するか、<https://docs.astral.sh/uv/getting-started/installation/> から手動で `uv` を入れて再試行してください。

**macOS：「開発元が未確認のため開けません」。**
`start-companion.command` を右クリック → **開く** → **開く**。最初の一回だけで大丈夫です。

**開くけれど返事がない／AI の頭脳が未設定と表示される。**
LLM の設定がまだです。セットアップウィザード（または 設定 → モデル）を開き、API キー（OpenAI / Claude / Gemini）を貼り付けるか、ローカルの Ollama モデルを選んでください。ローカルの場合は **Ollama アプリがインストールされ起動していること**、モデルがダウンロード済みであることを確認してください。

**Claude でウェブ検索／ツールが動かない。**
セットアップウィザードは Claude を互換エンドポイント経由でつなぐため、ツールが渡らず、実験的なウェブ検索／ツールのトグルは Claude では効かないことがあります。ツールを使うには GPT-4o や Gemini のようなツール呼び出し対応モデル、または能力のあるローカルモデルを使ってください。（Claude での通常のチャットは問題なく動きます。）
